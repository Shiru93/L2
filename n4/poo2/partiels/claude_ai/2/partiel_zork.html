        index = findNext(0);  // Positionner sur le 1er non-null
    }

    private int findNext(int from) {
        while (from < size && tab[from] == null) {
            from++;

        return from;  // Renvoie size si aucun non-null trouvé

public class FilterArrayIterator<E> implements Iterator<E> {

        index = findNext(index + 1);
        return elt;
    }

Exercice 4 Pattern Decorator — FilterIterator sur Room 5 points

On rappelle la classe Room (TP4) qui implémente Iterable<ObjetZork>, et la classe FilterIterator<E> du TD3, déjà complète :

FilterIterator.java — fournie (TD3) 
public class FilterIterator<E> implements Iterator<E> {
    private Iterator<? extends E> supportIter;
    private Predicate<? super E> predicate;
    private boolean nextFound;
    private E nextElt;

    public FilterIterator(Iterator<? extends E> iter, Predicate<? super E> p) {
        supportIter = iter; predicate = p; findNext();
    }
    private void findNext() {
        nextFound = false;
        while (supportIter.hasNext()) {
            E c = supportIter.next();
            if (predicate.test(c)) { nextElt = c; nextFound = true; return; }
        }
    }
    public boolean hasNext() { return nextFound; }
    public E       next()    { E r = nextElt; findNext(); return r; }
}
1Pourquoi dit-on que FilterIterator est un Decorator de Iterator ? Identifiez les deux éléments structurels du pattern dans ce code.1 pt

Correction

1. FilterIterator implémente la même interface (Iterator<E>) que l'objet qu'il décore → on peut l'utiliser partout où un Iterator est attendu.
2. Il contient une référence vers l'objet décoré (supportIter) et lui délègue les appels, en ajoutant un comportement (le filtrage).

2En utilisant FilterIterator, écrivez le code qui affiche le nom des objets transportables de la Room r (lambda obligatoire), puis enchaîne un second filtre gardant seulement les objets de poids > 5.2 pts

Correction

Le point clé : it1 est passé en paramètre à it2 → on empile les décorateurs.

public static Optional<Room> choisirPiece(Room current,

2Définissez la classe TakeObject implémentant Command : sa méthode execute() affiche les objets de la pièce courante via ListView, lit le choix du joueur, retire l'objet de la pièce et l'ajoute à l'inventaire du joueur.4 pts

Correction

public class TakeObject implements Command {
    private Gamer gamer;

    public TakeObject(Gamer gamer) {
        this.gamer = gamer;
    }

    @Override
    public void execute() {
        Room currentRoom = gamer.getCurrentRoom();

        // View : construire et afficher la liste
        ListView<ObjetZork> menu = ListView.buildMenu(currentRoom);
        menu.show();

        // View : lire le choix du joueur
        ObjetZork choix = menu.readChoice();

        // Model : modifier les données
        currentRoom.remove(choix);
        gamer.addToInventory(choix);

        // View : réafficher le menu principal
        menu.showMenu();
    }
}

Règle MVC respectée : seul le Controller (TakeObject) fait le lien. Le Model (Room, Gamer) n'appelle jamais la View.

Correction

public class RoomBuilder {
    String  shortName;
    String  description = "Une pièce";
    boolean hasTrap     = false;
    int     lightLevel  = 100;

    // Constructeur : paramètres obligatoires seulement
    public RoomBuilder(String shortName) {
        if (shortName == null) throw new NullPointerException();
        this.shortName = shortName;
    }

    // Méthodes optionnelles — renvoient this pour le chaînage
    public RoomBuilder withDescription(String d)  { description = d;    return this; }
    public RoomBuilder withTrap()                    { hasTrap = true;     return this; }
    public RoomBuilder withLightLevel(int l)         { lightLevel = l;     return this; }

    // build() : crée la Room finale
    public Room build() {
        return new Room(this);
    }
}
