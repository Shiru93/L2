1 Définissez la classe StrategieMaxTresors implémentant StrategieDeplacement. La méthode choisirSalle renvoie la salle voisine contenant le plus grand nombre de trésors satisfaisant le prédicat. Si aucune salle voisine n'existe, elle renvoie null. Appliquez également le pattern Singleton à cette classe. 4 pts

Correction

public class StrategieMaxTresors implements StrategieDeplacement {

    public static final StrategieMaxTresors INSTANCE = new StrategieMaxTresors();
    private StrategieMaxTresors() { }

    @Override
    public Salle choisirSalle(Salle p, Predicate<? super Tresor> pred) {
        Salle result = null;
        int max = 0;
        for (Direction d : Direction.values()) {
            Salle voisine = p.salleSuivante(d);
            if (voisine != null) {
                int count = 0;
                for (Tresor t : voisine) {     // possible grâce à Iterable
                    if (pred.test(t)) count++;
                }
                if (count > max) { max = count; result = voisine; }
            }
        }
        return result;  // null si aucune salle voisine
    }
}

Exercice 4 Design pattern Decorator — Itérateur filtrant les trésors rares 5 points

On dispose de la classe FilterIterator<E> suivante (vue en TD), qui décore n'importe quel Iterator pour ne garder que les éléments satisfaisant un prédicat :

FilterIterator.java — fournie

public class FilterIterator<E> implements Iterator<E> {
    private Iterator<? extends E> supportIter;
    private Predicate<? super E> predicate;
    private boolean nextFound;
    private E nextElt;

    public FilterIterator(Iterator<? extends E> iter, Predicate<? super E> p) {
        supportIter = iter; predicate = p; findNext();
    }
    private void findNext() {
        nextFound = false;
        while (supportIter.hasNext()) {
            E c = supportIter.next();
            if (predicate.test(c)) { nextElt = c; nextFound = true; return; }
        }
    }
    public boolean hasNext() { return nextFound; }
    public E next() { E r = nextElt; findNext(); return r; }
}
1 En utilisant FilterIterator, écrivez le code qui crée un itérateur parcourant uniquement les trésors rares d'une Salle s, sans modifier la classe Salle. Utilisez une lambda. 2 pts

Correction

Iterator<Tresor> iter = new FilterIterator<>(
    s.iterator(),
    t -> t.estRare()
);

C'est le pattern Decorator : FilterIterator enrobe l'itérateur de la salle sans en modifier le code.

2 Expliquez en 2-3 phrases pourquoi FilterIterator est un Decorator et pas une Strategy. En quoi les deux patterns diffèrent-ils ici ? 2 pts

Correction

FilterIterator est un Decorator car il implémente la même interface (Iterator<E>) que l'objet qu'il enrobe, et il lui délègue les appels en ajoutant un comportement (le filtrage). Il peut s'appliquer à n'importe quel itérateur.

La Strategy (Predicate) est ici utilisée à l'intérieur du Decorator pour paramétrer le critère de filtrage. La distinction clé : le Decorator modifie la façon dont on parcourt une structure ; la Strategy détermine le critère de sélection.

3 Un étudiant écrit Stream<Direction> s = Stream.of(Direction.values()); puis appelle s.filter(...) deux fois. Que se passe-t-il ? Pourquoi ? 1 pt

Correction

Une IllegalStateException est levée au deuxième appel. Un Stream est à usage unique : dès qu'une opération terminale (ou la première opération intermédiaire sur certaines implémentations) a été exécutée, le Stream est "consommé" et ne peut plus être utilisé. Il faut créer un nouveau Stream pour chaque chaîne de calcul.

1 Les classes suivantes appartiennent-elles au Model, à la View ou au Controller ? Justifiez en une phrase chacune. 1.5 pt

2 Définissez la classe RamassrTresor implémentant Command. Sa méthode execute() doit : afficher la liste des trésors de la salle courante via ListView, lire le choix du héros, puis retirer le trésor choisi de la salle et l'ajouter à l'inventaire du héros. Supposez que Salle implements Iterable<Tresor> et que Héros expose getSalleCourante() et ajouterInventaire(Tresor t). 3.5 pts

Correction

public class RamassrTresor implements Command {
    private Héros heros;

    public RamassrTresor(Héros heros) {
        this.heros = heros;
    }

    @Override
    public void execute() {
        Salle salle = heros.getSalleCourante();

        // View : construire et afficher la liste des trésors
        ListView<Tresor> menu = ListView.buildMenu(salle);
        menu.show();

        // View : lire le choix
        Tresor choix = menu.readChoice();

        // Model : retirer de la salle et ajouter à l'inventaire
        salle.retirerTresor(choix);
        heros.ajouterInventaire(choix);
    }
}

1 Définissez la classe HérosBuilder avec son constructeur, ses méthodes withXxx() et sa méthode build(). 4 pts

Correction

public class HérosBuilder {
    String  nom;
    int     pointsDeVie = 100;
    int     force       = 10;
    int     agilite     = 10;
    int     magie       = 0;
    boolean invincible  = false;

    // Constructeur : uniquement le paramètre obligatoire
    public HérosBuilder(String nom) {
        if (nom == null) throw new NullPointerException();
        this.nom = nom;
    }

    // Méthodes optionnelles — renvoient this pour le chaînage
    public HérosBuilder withPdv(int v)        { pointsDeVie = v; return this; }
    public HérosBuilder withForce(int v)      { force = v;       return this; }
    public HérosBuilder withAgilite(int v)    { agilite = v;     return this; }
    public HérosBuilder withMagie(int v)      { magie = v;       return this; }
    public HérosBuilder withInvincible()      { invincible = true; return this; }

    // build() : crée l'objet final
    public Héros build() {
        return new Héros(this);
    }
}
2 Donnez la déclaration minimale de la classe Héros nécessaire pour que HérosBuilder.build() fonctionne. Précisez en particulier la visibilité du constructeur de Héros et expliquez pourquoi. 2 pts

Correction

public class Héros {
    private final String  nom;
    private final int    pointsDeVie, force, agilite, magie;
    private final boolean invincible;

    // Package-private : ni public, ni private
    // Accessible depuis HérosBuilder (même package),
    // mais pas depuis n'importe quelle classe externe
    Héros(HérosBuilder b) {
        nom         = b.nom;
        pointsDeVie = b.pointsDeVie;
        force       = b.force;
        agilite     = b.agilite;
        magie       = b.magie;
        invincible  = b.invincible;
    }
    // ... getters ...
}

Le constructeur est package-private (sans modificateur de visibilité) pour que seul HérosBuilder (dans le même package) puisse créer un Héros. Pas de public pour forcer l'utilisation du Builder.
