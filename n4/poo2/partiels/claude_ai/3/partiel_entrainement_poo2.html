Contexte — Le Château Maudit

On développe un jeu de rôle textuel dans lequel un Chasseur explore un château hanté. Le château est constitué de Salles reliées entre elles par des passages (NORD, SUD, EST, OUEST). Chaque salle contient des Reliques : des objets anciens, certains maudits, d'une puissance variable.

Les classes suivantes sont fournies et ne sont pas à redéfinir : 

public class Relique {
    private String nom;
    private int puissance;
    private boolean estMaudite;

    public Relique(String nom, int puissance, boolean estMaudite) {
        this.nom = nom;
        this.puissance = puissance;
        this.estMaudite = estMaudite;
    }
    public String getNom()        { return nom; }
    public int getPuissance()     { return puissance; }
    public boolean estMaudite()   { return estMaudite; }
}

public enum Direction { NORD, SUD, EST, OUEST }

public interface Predicate<E> {
    boolean test(E e);
}

public interface Function<T, R> {
    R apply(T t);
}

    public void ajouterRelique(Relique r) {
        reliques[nbReliques++] = r;

    public Salle salleSuivante(Direction d) { /* fourni, peut renvoyer null */ }

    public Iterator<Relique> iterator() { /* À compléter — question 2 */ }
}
1. Définissez la classe SalleIterator qui implémente Iterator<Relique> et permet de parcourir le tableau reliques de taille nbReliques. Elle doit implémenter les méthodes hasNext() et next().
2. Complétez la méthode iterator() de la classe Salle.

Exercice 2 Design patterns Singleton & Enum 4 points
1. Définissez un enum Java TypeGardien avec les trois valeurs SQUELETTE, VAMPIRE et SPECTRE. Chaque valeur possède un attribut int dangerosité et une méthode getDangerosité(). Choisissez librement les valeurs de dangerosité (ex : SQUELETTE moins dangereux que VAMPIRE, SPECTRE le plus dangereux).
2. On souhaite une classe JournalChateau dont il ne doit exister qu'une seule instance dans tout le programme. Elle possède une unique méthode : 
void enregistrer(String message)  // affiche "[LOG] " + message

Définissez la classe JournalChateau en appliquant le design pattern Singleton. Votre implémentation devra :

  • rendre impossible la création d'une seconde instance depuis l'extérieur,
  • interdire explicitement le clonage,
  • exposer l'instance unique via un attribut public static final.
Ne pas oublier d'interdire le clonage (clone()) — c'est une condition nécessaire pour un vrai Singleton.
Exercice 3 Design pattern Strategy & Lambda expressions 5 points

Pour décider vers quelle salle se déplacer, le chasseur utilise une stratégie. L'interface suivante est fournie : 

public interface StrategieExploration {
    /**
     * Renvoie la salle voisine recommandée depuis la salle p,
     * en fonction d'un critère pred sur les Relique,
     * ou null si aucune salle voisine n'existe.
     */
    Salle choisirSalle(Salle p, Predicate<? super Relique> pred);
}
1. Définissez la classe StrategieMaxReliques implémentant StrategieExploration de façon à ce que choisirSalle renvoie la salle voisine contenant le plus grand nombre de reliques satisfaisant le prédicat pred. Si plusieurs salles sont à égalité, renvoyez n'importe laquelle. Si aucune salle voisine n'existe, renvoyez null.
Rappel — itérer sur les directions Direction.values() renvoie un tableau de toutes les valeurs de l'enum, utilisable dans un for amélioré. salleSuivante(d) peut renvoyer null.
2. Appliquez le design pattern Singleton à la classe StrategieMaxReliques. Justifiez en une phrase pourquoi le Singleton est approprié ici.
3. Donnez la lambda expression Java correspondant à chacun des prédicats suivants :
a) La relique est non maudite et de puissance strictement supérieure à 50.
b) La relique est non nulle et son nom commence par "Or" (rappel : String.startsWith(String prefix)).
c) La relique satisfait simultanément : non maudite, puissance > 20, et nom commençant par "A".
4. En une phrase, expliquez pourquoi Predicate<E> est une functional interface et ce que cela implique pour l'usage des lambdas.

public class Chasseur {

public class ChateauMap {

// VIEW
public class ListView<E> {
    public static <T> ListView<T> buildMenu(Iterable<T> items) { ... }

    public E readChoice()    { ... }  // attend et renvoie le choix de l'utilisateur

1. Définissez la classe PrendreRelique implémentant Command. Sa méthode execute() doit :
  1. Afficher la liste des reliques de la salle actuelle du chasseur.
  2. Lire le choix de l'utilisateur.
  3. Faire prendre la relique choisie par le chasseur.
