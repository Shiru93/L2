QCM d'administration système : les chemins

1 questions

1.1 question n° 1

Les chemins /.. et . désignent toujours le même répertoire

  • Faux
  • Vrai
  • On ne peut pas savoir
Indication : Par exemple, si le répertoire courant est /home, alors /.. (qui vaut /) et /home désignent des répertoires différents.

1.2 question n° 2

Les chemins /.. et . peuvent désigner le même répertoire

  • Vrai
  • Faux
  • On ne peut pas savoir
Indication : Si le répertoire courant est /, alors c'est le cas.

1.3 question n° 3

Les chemins /.. et . désignent le même répertoire

  • On ne peut pas savoir
  • Faux
  • Vrai
Indication : Ça dépend du répertoire courant. Si le répertoire courant est / alors /.. et . désignent tous les deux le répertoire /, mais si le répertoire courant est /home, alors /.. (qui vaut /) et /home désignent des répertoires différents.

1.4 question n° 4

Le répertoire courant est /. Les chemins /.. et . désignent le même répertoire

  • Vrai
  • Faux
  • On ne peut pas savoir
Indication : Le répertoire parent de / est / lui-même, si le répertoire courant est /, alors . designe aussi /.

1.5 question n° 5

Les chemins ../ et /.. désignent toujours le même répertoire

  • Faux
  • Vrai
  • On ne peut pas savoir
Indication : Par exemple, si le répertoire courant est /etc/apt/ alors ../ désigne /etc/ alors que /.. désigne /. Attention, un exemple de chemin de profondeur 1 comme /home ne fonctionne pas comme contre-exemple car dans ce cas ../ désigne aussi /.

1.6 question n° 6

Les chemins ./ et /. désignent toujours le même répertoire

  • Faux
  • Vrai
  • On ne peut pas savoir
Indication : Par exemple, si le répertoire courant est /home, alors ./ (qui vaut /home et /. (qui vaut /) sont différents.

1.7 question n° 7

Les chemins ./.. et ../. désignent toujours le même répertoire

  • Vrai
  • Faux
  • On ne peut pas savoir
Indication : Faire du sur place puis remonter revient au même que remonter puis faire du sur place.

1.8 question n° 8

Les chemins ~/.ssh/ et /home/user/.ssh/ désignent toujours le même répertoire

  • Faux
  • Vrai
  • On ne peut pas savoir
Indication : Le ~ est remplacé par le HOME de l'user courant. Le HOME de l'user root est /root pas /home/root. Aussi, on peut modifier la variable HOME comme on veut, ce qui modifie le ~ .

1.9 question n° 9

Les chemins ~ et ${HOME} désignent toujours le même répertoire

  • Vrai
  • Faux
  • On ne peut pas savoir
Indication : Voir man bash puis /Développement du tilde (le / permet de faire une recherche de motif)

1.10 question n° 10

Les chemins /~ et /${HOME} désignent toujours le même répertoire

  • Faux
  • Vrai
  • On ne peut pas savoir
Indication : Pour être remplacé par la valeur de la variable HOME, le tilde doit être au début du chemin et ne doit pas être protégé (par des guillemets). Voir man bash puis /Développement du tilde.

1.11 question n° 11

Le chemin ~ est un chemin absolu ?

  • On ne peut pas savoir
  • Vrai
  • Faux
Indication : Le chemin ~ dépend de la variable ${HOME} du shell courant. Il se pourrait (bien que ça soit invraissemblable), que la valeur de cette variable soit un chemin relatif.

1.12 question n° 12

Les fichiers de configuration du système se trouvent dans le répertoire /etc ?

  • Vrai
  • Faux
  • On ne peut pas savoir
Indication : Voir man 7 file-hierarchy ou man 7 hier .

1.13 question n° 13

Les fichiers de configuration du système se trouvent dans le répertoire ./etc ?

  • On ne peut pas savoir
  • Faux
  • Vrai
Indication : Les fichiers de configuration du système se trouvent dans le répertoire /etc qui est égal à ./etc seulement si le répertoire courant est /.

1.14 question n° 14

Les fichiers de configuration du système se trouvent dans le répertoire ~/etc ?

  • On ne peut pas savoir
  • Faux
  • Vrai
Indication : Les fichiers de configuration du système se trouvent dans le répertoire /etc qui est égal à ~/etc seulement si le HOME de l'user courant est /.
